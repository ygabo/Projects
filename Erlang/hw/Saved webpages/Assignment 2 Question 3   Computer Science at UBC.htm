key and value

Submitted by mrg on Sun, 2011-10-16 20:47.

Each worker has an association list to maintain state between tasks. Think of it as being like a java hashtable. Because erlang is functional, we have to pass the process state from one worker task to the next. To give a worker a task, you can broadcast the function for that task to all the workers using workers:broadcast(W, Task). Here W is a worker pool (returned by workers:create), and Task(ProcState1) -> ProcState2 is a function. The argument to Task is the association of keys to values established by everything done before invoking Task. Task is responsible for returning an updated association list.

For example, if sieve(N) is a function that returns the list of primes in [2, .. N], then if the root process executed:

?
1
workers:broadcast(W, fun(ProcState1) -> workers:put(ProcState1, 'first-primes', sieve(N)) end)

Each worker would have the list of primes in [2,.. N] associated with the key 'first-primes'.

You can give an additional parameter to workers:broadcast, a list with one element for each worker. You could then do something like:

?
1
2
3
4
5
6
7
NN = ...,
workers:broadcast(W, fun(ProcState1, M) ->
  Lo = M*NN + 1,
  Hi = (M+1)*NN,
  P1 = workers:get(ProcState1, 'first-primes'),
  workers:put(ProcState1, 'my-primes', find_primes(Lo, Hi, P1))
end, lists:seq(0, workers:n_workers(W)-1))

Then, later tasks could access the list of primes for the particular process by calling workers:get(ProcState, 'my-primes').

Have fun,
--Mark
